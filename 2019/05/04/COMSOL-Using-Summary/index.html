常见问题及解决办法

  1. RF 模型与 Wave Optics 模型的设置是不一样的，前者默认的是设置的介电常数等，后者默认的是要自己设置折射率。注意二者之间的转换。另外有时候会把材料的电导率 (Electrical conductivity) 写成了 1，一般应该取 0.
  2. 在 COMSOL 中，如果是为了增加损耗，应该在介电常数前面加相应的负的虚部. 如果是正虚部，就是引入了增益而不是损耗，这与电磁波的写法有关。我们物理学科一般写为 $e^{ikx-i\omega t}$ , 但是工科一般写为 $e^{-ikx+i\omega t}$ .
  3. PML 的作用是让向外辐射的场衰减而不会反射，利用的是介电常数的一些性质，但是仅仅是针对介电常数为正的材料，如果是负的，那么COMSOL默认的PML就不能很好的吸收，特别是遇到无穷长的金属需要深入到PML里面时，需要用另外一种UPML。
  4. 在对偶极子进行设计的时候，应该从偶极子开始，由内到外，画多个球体，设置不同的 mesh 大小，一步步的逐渐减少 mesh 的精度，设计多个层数进行变化，这样计算的精度会好很多。
  5. 一般来说，计算的时候，不需要自己去选择 Solver, 软件会根据实际的问题自己选择最优的 Solver, 不过需要注意的是，对于一些有谐振存在的情况，我们需要手动选择 direct 的 solver, 不然最后结果就会有问题。直接计算的求解器比迭代求解器在三维的情况下，所需要的 mesh 要稀疏，但是需要更大的内存来计算，并且速度较慢。
  6. 在采用本征值求解器计算模式时候，理论上采用的是边界置零的条件，但是在有些模式束缚的特别不好的时候，需要将边界扩大。保证结果的准确性。目前时边界置零还是需要PML，还很难说。。。后期搞清楚了再更新。
  7. 参数化扫描的时候生成的 table 没有值，处理办法：将所有的 table 删除，然后在probe 里面讲所需要得到的结果如 power 进行“update”，此时 table 会自动生成一个新的 probe table, 这时候手动建立一个 table, 然后再“parametersweep”里面选择刚才的 table 就行了，而且可以让 table 即保存，也输出到文件里面。
  8. 记录一个很傻逼的错误。1E4=10000，而不是 1000
  9. 为了使得结果准确，在计算之前，一定要再三确认结果是不是正确的！从参数设置、几何结构、材料、边界条件、mesh等慢慢的检查。不要讲速度，而是要讲效率！
  10. 变量名字不要设置为”lambda”，因为会和系统内部的命名空间冲突
  11. 针对7的一个补充说明，出现问题7的本质原因在于不同的probe都需要有一个table以及plot window.当本来就存在一些table,又增加了一些probe时，如果不特殊设置，这些probe往往会自动用同一个table，这种冲突导致最终table根本没有结果。为了解决这个问题，需要首先删除所有的table，然后一个个的给probe通过new指定table（将table设置为new之后，点击update）。只是table也还不够，plot window也需要同样的处理。
  12. 关于direct与iteration求解器的选择。在谐振比较明显的情况下，最好用Directsolver来计算，这样会比较准确。最近（20190612）在计算一个金属球附近有一个dipole的结构，需要计算的是偶极子的辐射功率，有两种方法计算，
    $\frac{dW}{dt}=\int_{S}\langle S \rangle \cdot n dS=\frac{\omega}{2}Im{d^{*}\cdot E(r_0)}$,一种需要积分，一种只需要dipole那一点的场的数据，根据理论这两个结果应该是一样的。不过用迭代求解器计算的结果是不一样的。
  13. 在使用mpheval或者mphevalpoint去提取某些区域的数据时，可以通过输入这些区域的数值（如point:18）来指定具体是哪一个几何区域，这种方法有一个坏处，在进行参数扫描且空间位置正在变化的时候，这些数值有可能会变，导致最后不是提取的自己想要的数据点。这是可以采用definition里面的“selection”选项。

使用技巧

dipole源设置

  1. 为了保证结果的准确性，应该在用一个较小的多层的球面包裹住偶极子，由内至外半径分别为1nm,5nm,50nm,因为偶极子是一个很小的点，必须得加这样一个过渡进来。这样设置的球面也可以用来计算辐射功率。
  2. 在采用对称性结构时，灵活运用PEC和PMC,当我们对空间进行切割的时候，验证偶极子方向切割要用PMC，垂直则用PEC.
